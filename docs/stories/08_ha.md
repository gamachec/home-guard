### Story 8 : auto-discover home assistant

Ajouter l'agent dans Home Assistant de manière propre et "native" en utilisant le **MQTT Discovery**.

Envoyer une seule fois, au démarrage de l'agent, un payload JSON sur les topics suivants :

**A. Pour le mode (Sélecteur) (en remplaçant client_id par la valeur dans la config) :**
*   **Topic :** `homeassistant/select/<client_id>/mode/config`
*   **Payload :**
```json
{
  "name": "Mode d'utilisation",
  "unique_id": "<client_id>_mode",
  "command_topic": "cmnd/<client_id>/mode",
  "state_topic": "stat/<client_id>/current_mode",
  "options": ["ACTIVE", "BLOCKED"],
  "device": {
    "identifiers": ["<client_id>"],
    "name": "<client_id>",
    "model": "HomeGuard",
    "manufacturer": "HomeGuard"
  }
}
```

**B. Pour le statut de connexion (Capteur binaire) :**
*   **Topic :** `homeassistant/binary_sensor/agent_pc/connectivity/config`
*   **Payload :**
```json
{
  "name": "Etat",
  "unique_id": "<client_id>_online",
  "device_class": "connectivity",
  "state_topic": "stat/<client_id>/status",
  "payload_on": "online",
  "payload_off": "offline",
  "device": { "identifiers": ["<client_id>"] }
}
```

**C. Pour la liste des apps (Capteur de texte) :**
*   **Topic :** `homeassistant/sensor/<client_id>/apps/config`
*   **Payload :**
```json
{
  "name": "Applications en cours",
  "unique_id": "<client_id>_running_apps",
  "state_topic": "stat/<client_id>/running_apps",
  "device": { "identifiers": ["<client_id>"] }
}
```
